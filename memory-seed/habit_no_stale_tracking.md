name: Keep tracking files current — zero drift
description: Update STATUS/tracking files immediately after each task; never let them drift from reality
type: feedback
---
Update the tracking files (STATUS and any task/plan/decision docs) right after each task, so they always reflect reality. Stale tracking is worse than no tracking — it actively misleads.

**Why:** The next session (and recall) trusts the tracking files. A "not built yet" label on shipped work, or a "done" on something broken, sends work in the wrong direction.

**How to apply:**
- After finishing a task, update STATUS in the same stretch of work — don't defer it.
- When you catch one stale claim, check sibling files for the same stale claim (drift usually clusters).
- Keep rows terse; detail belongs in session notes, not the tracking row.
