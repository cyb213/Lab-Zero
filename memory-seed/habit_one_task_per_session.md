---
name: One task per session
description: Ship one task, then wrap the session; don't batch several or auto-start the next from a queue
type: feedback
---
Do one task per work session, then wrap. Don't chain several tasks into one session, and don't auto-start the next backlog item just because it's there.

**Why:** One task per session keeps each change focused, reviewable, and cleanly committed — and leaves the user in control of what's next. Batching blurs what actually shipped, bloats the session, and makes mistakes harder to trace.

**How to apply:**
- Finish the task, wrap it (verify, update tracking, commit), and stop.
- At a continue-vs-wrap checkpoint, ask the user with a recommendation — don't silently push into the next task.
- Standing approval for a multi-step plan isn't a license to batch its steps; ship one, let the user start the next fresh.
